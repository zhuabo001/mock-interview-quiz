# 编码规范
1. 使用`typescript`时， 请为变量、函数等添加类型，方便使用typescript的lsp进行编译时语法校验
2. 请遵从主流的ts最佳实践规范
3. 当类型不确定时，优雅地使用范型

# 开发流程
1. 当用户要求你执行某个需求时，请向用户确认是否已经存在对应的计划文件(*-plan.md), 若不存在，请要求用户先确认开发计划
2. 有了开发计划以后，请浏览开发计划，并编写对应的测试用例(本项目可以使用vitest)，严格遵从TDD流程
3. 完成测试用例书写后, 找到 需求的计划文档 `*-plan.md`, 在同级目录下，创建对应的`*-progress.md`, 内部维护一个两列表格，列1是plan文档中的步骤编号以及名称， 列2是当前该步骤的完成状态， 状态有`未完成`, `进行中`和 `已完成`，初始值都是未完成
4. 任何编辑或者更新动作前先更改对应progress.md文档中的对应步骤的状态，开始实际编写代码前，先将状态更改为‘进行中’，完成后再将状态更改为‘已完成’
5. 上一个步骤的状态被标记为'已完成'后才可以进行下一步的需求的开发
