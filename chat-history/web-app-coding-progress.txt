     基金销售管理工具 — 页面组件开发实现方案

     一、Context

     当前项目已具备：
     - 技术骨架：Vite + React 19 + TypeScript + React Router v6 + TanStack Query
     + Zustand + Ant Design 6.x + @ant-design/charts + MSW
     - 数据层：完整的 Mock API handlers（products / clients / followups /
     dashboard）和 TypeScript 类型定义
     - Mock 环境：public/mockServiceWorker.js 已就位，enableMocking.ts 通过
     VITE_USE_MOCK=true 启用

     需要开发的部分：
     - 路由配置与页面入口
     - 5 个页面组件（Home / Dashboard / FundsList / ClientsList）
     - 2 个侧边详情 Drawer（FundDetail / ClientDetail）
     - 1 个公共侧边栏 Layout
     - API Hooks 层（封装 TanStack Query）
     - 样式重置与 AntD 主题定制

     十、关键数据流说明

     10.1 Fund Detail 双向关联

     1. 用户在产品货架页点击「查看」
     2. uiStore.openFundDrawer(id) 打开 Drawer
     3. Drawer 内部调用 useFundDetail(id) → GET /api/funds/:id
     4. MSW handler 反查 holdings 表，联表 clients 返回 holders 列表
     5. Drawer 渲染 fund 基本信息 + holders Table

     10.2 Client Detail 双向关联 + 跟进记录

     1. 用户在客户管理页点击「查看」
     2. uiStore.openClientDrawer(id) 打开 Drawer
     3. Drawer 同时调用：
       - useClientDetail(id) → GET /api/clients/:id（返回 client + holdings +
     salesPerson）
       - useFollowups(id) → GET /api/followups?clientId=id
     4. MSW handler 正查 holdings 联表 funds，返回 holdings 列表
     5. Drawer 渲染三段内容

     10.3 新增跟进记录

     1. 用户在 ClientDetailDrawer 点击「+ 新增跟进」
     2. 弹出 AntD Modal + Form（类型 select、内容 textarea、下一步 input、日期
     date-picker）
     3. 提交调用 useCreateFollowup() mutation
     4. MSW 写入内存 followupsStore，返回 201
     5. onSuccess 中 invalidateQueries(['followups', clientId])
     6. Drawer 内的跟进记录列表自动刷新

⏺ 根据之前的研究，我已经找到了根本原因。问题出在 G2 v5 的表达式解析器
  (parseOptionsExpr)。

  根因分析：

  G2 v5 有一个 EXPR_WHITE_LIST = ['style', 'encode', 'labels', 
  'children']。对于这些 key 下的所有字符串值，如果字符串以 { 开头且以 } 结尾，G2
   会尝试将其编译为表达式。

  在我的 Pie 图配置中：
  label={{ type: 'spider', content: '{name}: {percentage}' }}

  label 被 @ant-design/plots 转换为 labels（在白名单中），然后 content: '{name}:
   {percentage}' 这个字符串以 { 开头、以 } 结尾，G2 尝试将 name}: {percentage
  编译为表达式，遇到 } 字符时抛出 ExpressionError: Unexpected character: }。

  Line 图的 lineStyle 回调也不是 v2.x 的有效 prop。
