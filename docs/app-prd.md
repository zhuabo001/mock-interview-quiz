# 产品需求文档（PRD）— P0 交付范围

> 产品名称：基金销售管理工具（内部系统）
> 目标用户：渠道销售人员
> 交付形态：React 单页应用（SPA），Vercel 静态部署
> 版本：P0 MVP

---

## 一、产品定位与目标

### 1.1 业务背景

资管公司渠道销售团队日常需要：
- 快速查阅在售基金产品的关键指标
- 维护客户关系、记录跟进动态
- 了解「客户持有了哪些产品」和「产品被哪些客户持有」的双向关系
- 通过数据概览把握业务全貌

当前缺乏统一的内部工具，信息散落在 Excel、微信、邮件中，效率低且容易遗漏。

### 1.2 产品目标

为销售人员提供一个**一站式的信息查询与客户管理工具**，核心价值：
1. **信息聚合** — 产品、客户、持仓、跟进记录集中管理
2. **双向关联** — 从产品维度和客户维度都能快速定位关联关系
3. **业务洞察** — Dashboard 帮助销售快速了解产品结构和跟进活跃度

### 1.3 成功指标

- 销售人员能在 3 次点击内找到任意客户的持仓详情
- 新增跟进记录的操作路径不超过 2 步
- Dashboard 一眼能看出产品线结构和近期跟进趋势

## 二、信息架构

```
应用
├── 侧边栏导航
│   ├── 数据概览（Dashboard）    ← 默认首页
│   ├── 产品货架
│   └── 客户管理
├── 顶部栏
│   └── 系统标题
└── 内容区
    └── 根据当前路由渲染对应模块
```

### 路由规划

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 数据概览 | 默认首页，Dashboard |
| `/funds` | 产品货架 | 产品列表 + 筛选搜索 |
| `/funds/:id` | 产品详情 | 侧栏/抽屉形式展示 |
| `/clients` | 客户管理 | 客户列表 + 筛选搜索 |
| `/clients/:id` | 客户详情 | 侧栏/抽屉形式展示 |

## 三、功能需求

### 3.1 模块一：产品货架

#### 3.1.1 产品列表页（`/funds`）

**页面结构**

```
┌─────────────────────────────────────────────────┐
│  [搜索框]  [类型下拉筛选]  [状态下拉筛选]         │
├─────────────────────────────────────────────────┤
│  产品列表表格                                     │
│  列：名称 | 类型 | 最新净值 | 成立规模 | 状态 | 操作 │
│  ...                                            │
├─────────────────────────────────────────────────┤
│  分页器                                          │
└─────────────────────────────────────────────────┘
```

**功能点**

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-1.1 | 产品列表展示 | 必做 | 表格形式，展示核心字段：名称、类型、最新净值、净值日期、成立规模、当前规模、状态 |
| F-1.2 | 按名称搜索 | 必做 | 输入关键词，模糊匹配产品名称，支持实时搜索或回车搜索 |
| F-1.3 | 按类型筛选 | 必做 | 下拉选择：股票型 / 债券型 / 混合型 / 货币型 / 指数型，支持「全部」选项 |
| F-1.4 | 按状态筛选 | 必做 | 下拉选择：募集中 / 运作中 / 已清盘，支持「全部」选项 |
| F-1.5 | 分页 | 必做 | 默认每页 10 条，展示总条数和页码切换 |
| F-1.6 | 查看详情 | 必做 | 点击产品行或「查看」按钮，打开产品详情 |

**交互规则**
- 筛选条件变化时自动重新请求，页码重置为第 1 页
- 搜索框支持清空，清空后恢复全量数据
- 列表为空时展示空状态提示

#### 3.1.2 产品详情（抽屉/侧栏）

**页面结构**

```
┌─────────────────────────────────┐
│  产品名称                        │

├─────────────────────────────────┤

│  最低认购 | 产品描述              │
├─────────────────────────────────┤
│  持有人列表                      │
│  列：客户名 | 等级 | 持有金额 |    │
│      买入日期 | 收益率            │
└─────────────────────────────────┘
```

**功能点**

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-1.7 | 产品基本信息展示 | 必做 | 展示 Fund 实体的全部字段，合理分组排版 |
| F-1.8 | 持有人列表 | 必做 | 展示持有该产品的客户列表（反向关联），包含客户名、等级、持有金额、买入日期、收益率 |
| F-1.9 | 持有人跳转 | 必做 | 点击持有人姓名可跳转到对应客户详情 |

**功能点**

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-2.6 | 客户基本信息展示 | 必做 | 展示 Client 实体全部字段 + 负责业务员信息 |

#### 3.2.3 新增跟进记录表单

**表单字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 跟进方式 | 下拉选择 | 是 | 电话 / 面谈 / 微信 / 邮件 |
| 跟进内容 | 多行文本 | 是 | 本次跟进的具体内容 |
| 下一步计划 | 单行文本 | 是 | 下一步要做什么 |
| 下一步日期 | 日期选择器 | 是 | 计划执行日期 |

**交互规则**
- 提交成功后关闭表单，跟进记录列表自动刷新
- 提交失败时展示错误提示，表单内容不丢失
- `clientId` 和 `salesPersonId` 由系统自动填入，用户无需手动选择

### 3.3 模块三：数据概览（Dashboard）

#### 3.3.1 Dashboard 页面（`/`）

**页面结构**

```
┌─────────────────────────────────────────────────┐
│  统计卡片区                                       │
│  [产品总数] [在售产品] [客户总数] [管理总规模]       │
├────────────────────────┬────────────────────────┤
│  图表一：产品规模占比     │  图表二：跟进趋势        │
│  （环形图/饼图）         │  （折线图）             │
│                        │                        │
│                        │                        │
└────────────────────────┴────────────────────────┘
```

**功能点**

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-3.1 | 概览数字卡片 | 必做 | 4 个统计卡片：产品总数、在售产品数、客户总数、管理总规模（格式化为亿元） |
| F-3.2 | 产品规模占比图 | 必做 | 环形图/饼图，展示在售产品按类型分组的规模占比，hover 显示具体数值 |
| F-3.3 | 跟进趋势图 | 必做 | 折线图，展示近 6 个月的跟进次数趋势，支持按跟进方式细分（多条折线） |

**图表选型理由**
- 规模占比图：销售最关心「公司产品线结构是否均衡」「哪类产品规模最大可以重点推」
- 跟进趋势图：跟进频次直接反映团队活跃度，按方式细分能看出触达策略偏好

## 四、全局交互规范

### 4.1 布局

- 左侧固定侧边栏导航（宽度 200px），包含 3 个一级菜单项
- 顶部栏展示系统名称
- 右侧为内容区，自适应宽度
- 最小支持宽度 1280px（内部系统，不要求移动端适配）

### 4.2 导航

- 侧边栏当前页面高亮
- 详情以 Drawer（抽屉）形式从右侧滑出，不离开当前列表页
- 抽屉内的跳转链接（如持有人→客户详情）关闭当前抽屉后打开新抽屉

### 4.3 数据加载

- 所有数据通过异步请求获取（MSW 拦截）
- 列表加载时展示 Loading 状态（骨架屏或 Spin）
- 请求失败时展示错误提示，提供重试按钮

### 4.4 数据格式化

| 数据类型 | 格式化规则 | 示例 |
|----------|-----------|------|
| 金额（元） | 万元/亿元自动换算，保留 2 位小数 | `50000000` → `5,000.00 万` |
| 净值 | 保留 4 位小数 | `1.4823` |
| 收益率 | 百分比，保留 2 位小数，正数绿色负数红色 | `0.3725` → `+37.25%` |
| 日期 | `YYYY-MM-DD` | `2025-03-14` |
| 规模 | 亿元，保留 2 位小数 | `2850000000` → `28.50 亿` |
