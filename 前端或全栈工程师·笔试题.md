## 场景背景

你所在的资管公司需要一个内部 Web 工具，帮助渠道销售团队管理基金产品信息和客户关系。

你的用户是 渠道销售人员 ，他们的核心诉求是：

- 快速了解公司在售基金产品的关键信息（类型、净值、规模、状态等）
- 维护客户关系，记录跟进情况
- 快速查询"某个客户买了哪些产品"或"某个产品被哪些客户持有"

## 功能要求

请围绕以下 三个核心模块 进行设计和实现：

### 模块一：产品货架

基金产品的展示与管理。至少应包含：

- 产品列表，支持按关键维度（如产品类型、状态）进行筛选和搜索
- 产品详情的查看方式（弹窗、侧边栏、独立页面等，你来决定）
- 产品数据需包含但不限于：名称、类型（股票型/债券型/混合型等）、最新净值、成立规模、产品状态（募集中/运作中/已清盘等）

### 模块二：客户管理

客户信息与跟进记录。至少应包含：

- 客户列表与基本信息
- 每个客户关联了哪些产品（持有关系）
- 客户跟进记录的查看和新增（如：2025-03-01，电话沟通，讨论了XX产品认购意向）

### 模块三：数据概览

一个 Dashboard 页面，包含 至少两个有意义的数据可视化 （图表类型自选），帮助销售团队快速了解业务全貌。

图表内容由你决定——什么数据对销售团队最有价值？这是我们评估业务理解力的重要依据。

## ⭐ 重要加分项：引入 Agent 能力

如果你能在项目中集成 AI Agent 能力，将获得显著加分。

我们希望看到的不是简单的"套一个聊天框"，而是 **Agent 与业务系统的深度结合** ——AI 能理解用户意图，并直接操作系统中的数据和界面。

以下是一些可能的方向（仅供启发，不限于此）：

- **自然语言驱动 Dashboard** ：用户通过对话描述需求（如"帮我生成一个按产品类型分组的规模占比饼图"），Agent 动态生成对应的可视化图表
- **智能查询** ：用户用自然语言提问（如"张总持有哪些债券型产品？""上个月新增了几个客户？"），Agent 查询数据并返回结构化结果
- **辅助录入** ：用户用自然语言描述跟进内容（如"今天和李总电话聊了半小时，他对稳健增长一号有兴趣，打算下周认购50万"），Agent 自动解析并填充表单字段

实现方式不限： 可以接入真实 LLM API（OpenAI / Claude / 开源模型等），也可以用 Mock 模拟 Agent 的输入输出。我们主要评估的是：你如何设计 Agent 与现有系统的交互架构，以及这种集成带来的用户体验提升。

如果使用了真实 API，请确保 Demo 可运行，并在 **DESIGN.md** 中说明接入方式。如果 API Key 有成本顾虑，Mock 方案完全可以接受。

## 数据说明

请自行设计 Mock 数据结构和内容。 **数据建模能力是评分项之一** 。

数据量要求：不少于 **8 个产品 + 8 位客户** ，确保足以支撑筛选、搜索等功能的演示。

数据获取方式请模拟 **异步请求** （setTimeout / MSW / json-server 等均可），不要在组件中直接 import 数据。

如果你有全栈能力，搭建一个简单的后端服务会是加分项（非必须）。

这份文档的权重与代码同等重要。

请包含以下内容：

3. **AI 协作日志**

   - 当前实现中有哪些你已知但来不及修的问题？
   - 如果这是真实的企业项目，你会做哪些不同的决策？
